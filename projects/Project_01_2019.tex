\section{Introdução}

O objetivo deste projeto é identificar quais clientes de uma instituição financeira irão executar uma determinada transação no futuro. Os dados disponíveis para este projeto são dados reais de uma instituição financeira. No entanto, por questões de sigilo, todos os dados sensíveis foram anonimizados. 

\section{Dados}

Para a execução deste projeto você deverá utilizar três arquivos CSV: 

\begin{itemize}	
	\item arquivo com variáveis independentes para treinar o modelo: data/transacoes\_financeiras\_x\_treinamento.csv
	\item arquivo com variável dependente para treinar o modelo: data/transacoes\_financeiras\_y\_treinamento.csv
	\item arquivo de teste com as variáveis independentes: 
	data/transacoes\_financeiras\_x\_teste.csv
\end{itemize}


\section{Método de trabalho}

Sugere-se seguir as seguintes etapas para a execução desta análise:

\begin{itemize}
\item Aquisição, análise exploratória e pré-processamento dos dados; 
\item Construção do modelo;
\item Análise do modelo;
\item Preparação do relatório com as estimativas de resultados, e;
\item Geração do arquivo com os resultados para os exemplos no conjunto de teste cego (validação final). 
\end{itemize}

\section{Itens que devem ser entregues}

Os seguintes itens devem ser entregues:

\begin{itemize}
\item Um relatório apresentando os principais resultados da
  análise. Este relatório deve ser entregue na forma de um Notebook Python.
\item O conteúdo do mesmo deve possuir os seguintes itens:
\begin{itemize}
\item introdução, com descrição do contexto e objetivo do trabalho;
\item descrição do método utilizado para alcançar o objetivo, e;
\item conclusão, ou seja, descrição dos resultados encontrados com a
  análise realizada. 
\end{itemize}
\item O relatório deve apresentar os valores de F1-Score obtidos pelos modelos testados. Além de
apresentar os valores de F1-Score, a equipe deverá explicar qual é a equação e o conceito por trás desta
equação. Além disso, deve também explicar o motivo de utilizar F1-Score e não acurácia. 
\item Todos os comandos que fazem parte do script (aquisição, transformação, 
criação do modelo, avaliação, etc) devem estar presentes no Notebook Python. 
\item Arquivo texto com as predições para o arquivo de teste (data/transacoes\_financeiras\_y\_teste.csv)
\end{itemize}

\section{Equipes e prazos}

Este trabalho poderá ser feito em equipes com até três (3) integrantes e deverá
ser apresentado no dia 23 de outubro de 2019. 

\section{Avaliação}

A avaliação por equipe será composta pelos seguintes atributos:

\begin{enumerate}
	\item Relatório (60\% nota): o relatório precisa estar completo e correto. Além disso, precisa fazer uso de comentários em markdown para facilitar a leitura. Deve ser objetivo e claro. A nota do relatório pode ser entre 0 e 10.  
	\item Apresentação (20\% da nota): a equipe deve apresentar os resultados de forma clara e objetiva, fazendo uso do python notebook para discutir os resultados com os demais alunos da disciplina. A nota da apresentação pode ser entre 0 e 10. 
	\item Resultado na forma do arquivo com as predições (20\% da nota): 
	\begin{equation}
	nota\_predicoes = F1\_score* 0.4
	\end{equation}
\end{enumerate}

A nota final do relatório é calculado por:

\begin{equation}
nota\_final = \frac{6 * relatorio + 2 * apresentacao + 10 * nota\_predicoes}{10}
\end{equation}

Se nenhum dos itens for entregue a nota será zero (0).
